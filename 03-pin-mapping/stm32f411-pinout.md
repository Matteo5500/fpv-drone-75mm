# STM32F411CEU6 — Pinout and Functional Blocks

## Overview

The STM32F411CEU6 is an ARM Cortex-M4 microcontroller manufactured
by ST Microelectronics. It is the brain of the BetaFPV Meteor75 Pro
AIO Flight Controller, running Betaflight firmware.

### Core Specifications
| Parameter | Value |
|-----------|-------|
| CPU | ARM Cortex-M4 @ 100MHz |
| FPU | Yes (hardware floating point) |
| RAM | 128KB SRAM |
| Flash | 512KB |
| Package | QFN48 (48 physical pins) |
| Operating voltage | 3.3V (I/O) |

### Why the FPU Matters
PID calculations require floating point arithmetic (e.g., P=45.5,
I=80.2). Without a hardware FPU, these calculations would be
emulated in software — significantly slower. The Cortex-M4 FPU
executes them in a single clock cycle.

---

## GPIO and Alternate Functions

Each GPIO pin can be configured as:
- Digital input/output
- **Alternate Function (AF)** — maps the pin to an internal
  hardware peripheral (UART, SPI, timer, ADC)

This means a single physical pin can serve as a UART TX line,
a SPI clock, or a timer output — depending on software configuration.

## Peripheral Blocks Available

### UART (Serial Communication)
| Peripheral | TX Pin | RX Pin | Assigned To |
|------------|--------|--------|-------------|
| USART1 | PA9 | PA10 | ExpressLRS RX (CRSF @ 420000 baud) |
| USART2 | PA2 | PA3 | VTX SmartAudio (@ 115200 baud) |
| USART6 | PC6 | PC7 | Available |

### SPI (High-Speed Synchronous)
| Peripheral | SCLK | MISO | MOSI | Assigned To |
|------------|------|------|------|-------------|
| SPI1 | PA5 | PA6 | PA7 | ICM-42688-P IMU @ 24MHz |
| SPI2 | PB13 | PB14 | PB15 | Flash W25Q (Blackbox) @ 21MHz |

### Timers (DSHOT Signal Generation)
| Timer | Channel | Pin | Motor |
|-------|---------|-----|-------|
| TIM1 | CH1 | PA8 | Motor 1 |
| TIM4 | CH1 | PB6 | Motor 2 |
| TIM4 | CH2 | PB7 | Motor 3 |
| TIM4 | CH3 | PB8 | Motor 4 |

> DSHOT is generated via hardware timers, not software.
> This is critical: software-generated DSHOT would consume
> CPU cycles; hardware timers run independently.

### ADC (Analog to Digital Converter)
| Pin | Function |
|-----|----------|
| PA0 | VBAT voltage sensing (via voltage divider) |

### Debug Interface (SWD)
| Pin | Function |
|-----|----------|
| PB3 | SWD_CLK |
| PA13 | SWD_DIO |

Used for firmware flashing and debugging via ST-Link programmer.

## Key Design Constraints

1. **UART scarcity:** Only 3 UARTs available. Each serial
   peripheral (RX, VTX, GPS) requires an exclusive UART.
   Currently 2/3 UARTs are allocated.

2. **Timer allocation for DSHOT:** Each motor requires a
   dedicated hardware timer channel. With 4 motors on TIM1
   and TIM4, no timer channels remain for additional DSHOT outputs.

3. **SPI bus separation:** IMU and Flash use separate SPI buses
   (SPI1 and SPI2). This avoids bus contention — the IMU
   is read at 8kHz while Blackbox writes at lower frequency.

4. **Hardware DSHOT:** DSHOT signals are generated by hardware
   timers, not by the CPU. This ensures precise timing
   regardless of CPU load.
